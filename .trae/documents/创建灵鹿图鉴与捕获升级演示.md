# 创建“灵鹿图鉴”页面与捕获升级系统

## 1. 核心功能设计

我们将创建一个全新的“灵鹿图鉴”页面，集成了宝可梦式的收集体验、养成互动以及基于地理位置（GPS）的记录功能。

### 1.1 图鉴页面 (Pokedex)

* **视觉风格**: 游戏化UI，采用卡片式布局。未获得的灵鹿显示剪影（阴影），已获得的显示全彩精美立绘（和小程序主页面风格一致）。

* **稀有度系统**: 分为 N (普通), R (稀有), SR (史诗), SSR (传说)。不同稀有度有不同的边框特效。

* **详情展示**:

  * **动态立绘**: 支持简单的呼吸/悬浮动效。

  * **个性化故事**: 每只被捕获的小鹿都有独一无二的背景故事（基于AI随机生成）。

  * **情绪感知**: 显示当前小鹿的心情（开心、疲惫、期待等）。

  * **捕获地点**: 记录捕获时的真实经纬度，并真实地名（如“迷雾森林 (31.23, 121.47)”）。

  * **运势加成**: 不同的鹿提供不同的运势Buff（如“桃花运 +10%”）。

### 1.2 互动演示 (Demo)

* **捕获/孵化演示**: 在图鉴页增加一个“探索捕获”按钮，孵化小鹿水晶球（演示用）。点击后模拟扫描、发现、捕获的过程，伴随震动和特效，随机获得一只灵鹿。

* **升级演示**: 在详情页增加“注入灵力”按钮。点击后播放升级特效，等级提升，属性增强。

### 1.3 入口设置

* 在首页 (`SJShouYe`) 的“我的守护兽”区域或功能区添加“图鉴”入口，点击跳转至新页面。

## 2. 技术实现方案

### 2.1 文件结构

* **新页面**: `src/pages/Pokedex/index.jsx` (逻辑) 和 `src/pages/Pokedex/index.module.scss` (样式)。

* **配置**: 更新 `src/app.config.js` 注册新页面路由。

### 2.2 数据结构设计

我们将定义一套灵鹿的数据模型，存储在本地缓存 `Taro.getStorageSync('caught_deers')` 中。

```javascript
{
  id: "deer_001",
  name: "森之灵鹿",
  rarity: "R",
  level: 1,
  capturedAt: "2023-12-31",
  location: { latitude: 30.1, longitude: 120.2, name: "xx公园" },
  story: "它是在雨后的清晨被发现的...",
  mood: "happy",
  fortune: "今日宜出行"
}
```

### 2.3 关键逻辑

* **GPS定位**: 使用 `Taro.getLocation` 获取当前位置，用于记录捕获/孵化点。

* **Mock数据**: 预设约 6-8 种不同种类的灵鹿配置（图片、基础描述、稀有度），捕获时从中随机抽取并生成个性化属性。

## 3. 执行步骤

1. **创建页面文件**: 编写 `src/pages/Pokedex/index.jsx` 和样式文件，实现网格列表和详情弹窗 UI。
2. **实现捕获逻辑**: 编写随机生成灵鹿、获取GPS、保存数据的函数。
3. **实现升级逻辑**: 编写消耗灵力升级的函数。
4. **添加入口**: 修改首页 `src/pages/SJShouYe/SJShouYe.jsx`，加入跳转按钮。
5. **注册路由**: 更新 `src/app.config.js`。
